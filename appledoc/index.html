    A Note on Usage

    All functionality should move through ETPush and it’s methods. ETPush is a singleton for your convenience, and will handle communication with Salesforce automatically. There are a few internal use methods exposed, but should not be used and are documented as such.

    Version 3.0 of the Journey Builder for Apps SDK brings new features offered by Salesforce, most notably the introduction of location services, starting with Geofenced messages. Location Services are enabled through the combination of two settings - first, the SDK must be set up for Location at configuration time ETPush’s configuration method, along with explicitly instructing the LocationManager to start watching location . By having a two-key system like this, your app can (and should strongly consider) allowing the user to turn on these features in the app itself, and provide a switch or button to control that. Note that both are required for Location services to work. Finally, the user has ultimate control over Location Services through the Settings app provided by iOS. The LocationManager will check for permissions on startWatchingLocation (and subsequent relevant times), and must respect what iOS tells it through those methods.

    More information is available at Code@ExactTarget.
